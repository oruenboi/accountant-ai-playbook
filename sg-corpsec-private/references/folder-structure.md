# Suggested Folder Structure for SG CorpSec Clients

Root per client (example): CorpSec/{{COMPANY_NAME}}/

01-BizProfile-and-IDs/
- Latest BizProfile.pdf, UEN letter, constitution, KYC IDs, address proofs.

02-Registers/
- RORC/, ROND/, RONS/, Members/, Directors-CEO/, Charges/, Seal/
- Keep signed controller/nominee declarations and working copies of lodgements.

03-Board-Resolutions/
- YYYY/ folder per year; file minutes and written resolutions (e.g., 2026/2026-02-Board-AGM-Waiver.md).

04-Members-Resolutions/
- YYYY/ folder; include special/ordinary resolutions and execution evidence.

05-Share-Capital/
- Allotments/, Transfers/, Buybacks-Redemptions/, Certificates/
- Place stamping certs, return of allotment receipts, share certificate PDFs/logs.

06-Financials-and-AGM/
- FY{{YYYY}}/FS.pdf, AGM/waiver packs, AR screenshots/transaction receipts.

07-Filings-and-Receipts/
- BizFile+/IRAS/MOM acknowledgements; name them 2026-03-AR-Txn123456.pdf.

08-Contracts-and-Banks/
- Bank mandate changes, authorised signatory lists, counterpart notices (address/name change letters).

09-Compliance-Working-Papers/
- Checklists, gap analyses, remediation plans; keep the live compliance tracker here (e.g., `compliance-matters-tracker.md` per company).

10-Misc/
- Anything not covered; periodically triage into the right bucket.

Naming conventions
- Prefix files with ISO date: YYYY-MM-DD-<short-description>.<ext>.
- Use hyphens, avoid spaces; keep names under 120 chars.
- For multiple versions, suffix -v1, -v2, but keep the signed/executed version clearly labeled -signed.

Access and retention
- Keep minute books and statutory registers in 02-Registers; mirrors can exist in DMS but mark the canonical location.
- Retain accounting/transaction support 5 years; keep corporate records permanently.
- Store evidence of filings (BizFile+ transaction numbers) alongside the related resolution.
